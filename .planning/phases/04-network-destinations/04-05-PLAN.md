**1. Add DestinationManager as an observable property** (near the top, after `var watchFolders`):
```swift
/// Per-destination availability state. Exposed for Settings and MenuBar UI.
var destinationManager = DestinationManager()
```

**2. Update BackupEngine init in setup()** — BackupEngine must be initialized with all adapters. Replace:
```swift
let adapter = LocalDestinationAdapter(config: dest)
self.engine = BackupEngine(db: database, adapters: [adapter])
```
With the approach of loading all DestinationConfig rows from GRDB and building adapters via DestinationManager, then initializing BackupEngine. However, BackupEngine.adapters is set at init time and is private. So the correct approach is:

- After DestinationManager is set up with all configs, get the adapters from it
- Initialize BackupEngine with those adapters

```swift
// Load all destination configs from DB
let allConfigs = try await database.pool.read { db in
    try DestinationConfig.order(Column("createdAt").asc).fetchAll(db)
}

// Set up DestinationManager with all configs
destinationManager.setup(configs: allConfigs)

// Build adapters list for BackupEngine initialization
// Use DestinationManager's factory — adapters keyed by ID in destinationManager
// BackupEngine needs [any DestinationAdapter] at init time
let engineAdapters = allConfigs.compactMap { config in
    destinationManager.adapter(for: config.id)
}
self.engine = BackupEngine(db: database, adapters: engineAdapters.isEmpty ? [LocalDestinationAdapter(config: dest)] : engineAdapters)
```

The bootstrap local destination (`dest`) is still saved to GRDB (upsert) before this point — keep that logic. This ensures new installs have at least one destination.

**3. Update runBackup() to dispatch to available destinations**

Replace the Phase 2/3 bootstrap job dispatch block:
```swift
// OLD:
let job = BackupJob(project: project, destinationIDs: [bootstrapDestID])
_ = try await engine.runJob(job)
```

With:
```swift
// Phase 4: dispatch to all available destinations
let available = destinationManager.availableAdapters()
let availableIDs = available.map(\.id)

if availableIDs.isEmpty {
    // No destinations available — surface as error
    status = .error("No destinations are currently available")
    NotificationService.sendBackupFailure(projectName: projectName, error: "No destinations available")
    return
}

let job = BackupJob(project: project, destinationIDs: availableIDs)
let result = try await engine.runJob(job)

// Check for per-destination failures (partial success)
for destResult in result.destinationResults {
    if let errorMsg = destResult.errorMessage {
        logger.warning("runBackup: destination \(destResult.destinationID) failed — \(errorMsg, privacy: .public)")
        // Update DestinationManager status for failed destination
        destinationManager.statuses[destResult.destinationID] = .unavailable(reason: errorMsg)
    }
}
```

Keep all other runBackup() logic (status management, notification calls, error handling) unchanged.

**IMPORTANT**: Do not remove the `bootstrapDestID` and `bootstrapProjectID` constants — they are still used for the upsert logic in setup(). Only the job dispatch changes.
