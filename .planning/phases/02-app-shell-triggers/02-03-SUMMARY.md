## Files Created/Modified

- `AbletonBackup/SchedulerTask.swift` - @MainActor final class; start() cancels existing Task before spawning new loop; Task.sleep(for:) cancellation handled with do/catch; deinit cancels task

## Decisions Made
- NotificationService uses static methods (not singleton) — stateless service with no lifecycle to manage
- SchedulerTask is @MainActor-isolated because stored Task<Void, Never> is non-Sendable; this prevents Swift 6 concurrency errors without any workarounds
- LoginItemManager.isEnabled is a computed property reading SMAppService live — any caching approach (UserDefaults, @State) would desync with System Settings changes
- ServiceManagement framework imported without any additional linker flags or entitlements — works as expected for non-sandboxed distribution
- xcodegen regeneration required after adding new Swift files (project.yml sources `AbletonBackup` directory but xcodeproj must be regenerated to pick up new files in build phases)
