**Root cause:** Two issues prevent notifications from appearing:
1. The `getNotificationSettings` guard skips `requestAuthorization` for any status other than `.notDetermined`. On an unsigned debug build run from DerivedData, the notification center may not have a stable identity for the app — the status may appear as `.notDetermined` but `requestAuthorization` never shows a dialog, or the status is `.denied` from a previous misidentified bundle. The fix: call `requestAuthorization` directly without the `.notDetermined` guard, then log the result via the Logger added in plan 02-06.
2. No `UNUserNotificationCenterDelegate` is set. Without a delegate, notifications delivered while the app is in the foreground (as a menu bar app, it is always "in the foreground") are silently suppressed by UNUserNotificationCenter — the system delivers them to the delegate's `userNotificationCenter(_:willPresent:)` method, and if no delegate is set, they are dropped.

**Changes to NotificationService.swift:**

Replace the existing `requestAuthorization()` method with a new `setup()` static method that both sets the delegate and requests authorization:

```swift
// MARK: - Setup (call once at app launch)

/// Sets the UNUserNotificationCenter delegate and requests authorization.
/// Must be called at app launch before any notification is posted.
/// Without a delegate, foreground notifications are silently suppressed.
static func setup() {
    UNUserNotificationCenter.current().delegate = NotificationDelegate.shared
    UNUserNotificationCenter.current().requestAuthorization(
        options: [.alert, .sound]
    ) { granted, error in
        notifLogger.info("requestAuthorization: granted=\(granted) error=\(error?.localizedDescription ?? "nil", privacy: .public)")
    }
}
```

Add a `NotificationDelegate` class at the bottom of `NotificationService.swift`:

```swift
// MARK: - NotificationDelegate

/// Ensures notifications display as banners while the app is in the foreground.
/// Menu bar apps are always in the foreground — without this delegate,
/// UNUserNotificationCenter silently drops all notifications.
final class NotificationDelegate: NSObject, UNUserNotificationCenterDelegate {
    static let shared = NotificationDelegate()
    private override init() {}

    func userNotificationCenter(
        _ center: UNUserNotificationCenter,
        willPresent notification: UNNotification,
        withCompletionHandler completionHandler: @escaping (UNNotificationPresentationOptions) -> Void
    ) {
        // Show banner + play sound even when app is in foreground
        completionHandler([.banner, .sound])
    }
}
```

Keep `sendBackupSuccess`, `sendBackupFailure`, and `post` methods unchanged.

Remove `requestAuthorization()` entirely (replaced by `setup()`).

Replace the two silent guard returns in `runBackup(trigger:)` with error-surfacing paths:

```swift
// Before (silent):

    logger.warning("runBackup: guard — already running, ignoring \(String(describing: trigger), privacy: .public) trigger")
    return
}
```
(The logger call was already added in plan 02-06. This guard is correct behavior — just needs the log that plan 02-06 added. No user-visible change needed here; a concurrent trigger while running is not an error.)

**Guard 2 — not configured:**
```swift
// Before (silent):

// After (surfaces error state):
guard let engine, let db, let watchedProjectsFolder else {
    let reason: String

    logger.error("runBackup: guard — not configured: \(reason, privacy: .public)")
    status = .error(reason)
```
