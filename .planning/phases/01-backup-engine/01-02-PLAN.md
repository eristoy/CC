Incremental skip heuristic — mtime + size comparison:
```swift
// Returns true if file should be copied; false if it can be skipped
func needsCopy(entry: FileEntry, previousRecord: BackupFileRecord?) -> Bool {
    guard let prev = previousRecord else { return true }   // new file
    return entry.size != prev.sourceSize || entry.mtime > prev.sourceMtime
}
```
NOTE: Do NOT compute source-side checksum to decide whether to copy — that reads every byte of every file and eliminates incremental benefit. mtime+size for skip decisions only.

  <name>ProjectResolver — folder walker with incremental skip heuristic</name>
  <files>Sources/BackupEngine/ProjectResolver.swift, Tests/BackupEngineTests/ProjectResolverTests.swift</files>
  <behavior>
    Given: a directory URL on disk
    Returns: [FileEntry] for all non-hidden, non-directory files, recursively

    Inclusion rules:
    - Include: regular files at any depth
    - Exclude: directories (only files in results)
    - Exclude: hidden files (names starting with ".")
    - Include: .als files (Ableton project file — not hidden despite starting with no dot)
    - relativePath: path relative to the root directory, using "/" separator, no leading slash
      e.g., root=/tmp/MyProject, file=/tmp/MyProject/Samples/kick.wav → relativePath="Samples/kick.wav"

    Incremental skip heuristic:
    - needsCopy(entry:previousRecord:) where previousRecord is nil → true (new file)
    - needsCopy with matching size AND mtime ≤ prev.sourceMtime → false (skip)
    - needsCopy with different size (same mtime) → true (changed)
    - needsCopy with newer mtime (same size) → true (changed)

    Test cases (all must be in the test suite):
    - Empty directory → empty array
    - Flat directory with 3 files → 3 FileEntry values with correct relativePaths
    - Nested directory → all files returned with correct relative paths
    - Hidden file (.DS_Store) excluded
    - Directory entry excluded from results (only files)
    - FileEntry.size matches actual byte count of file
    - FileEntry.mtime matches file's modification date
    - needsCopy: nil previousRecord → true
    - needsCopy: matching size and mtime → false
    - needsCopy: different size → true
    - needsCopy: newer mtime → true
  </behavior>
  <implementation>
    Use FileManager.enumerator(at:includingPropertiesForKeys:) with keys:
    [.isRegularFileKey, .isHiddenKey, .fileSizeKey, .contentModificationDateKey]

    Filter by isRegularFile == true and isHidden == false.

    Compute relativePath by stripping the root directory prefix from the file URL's path.

    All operations are synchronous (no async needed — local file system only).
    Mark ProjectResolver as a struct (not actor) — it is stateless.
  </implementation>
</feature>
