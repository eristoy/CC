  <what-built>
    Complete Phase 2 app: menu bar utility with FSEvents save detection, scheduled backup, manual trigger, macOS notifications, and login item support.

    The app should be running now (launched in Task 1). Verify these 7 behaviors:
  </what-built>
  <how-to-verify>
**Step 1 — No Dock icon (APP-01)**
Look at your Dock. AbletonBackup should NOT appear there.
Expected: No AbletonBackup icon in Dock, no AbletonBackup in Cmd+Tab app switcher.

**Step 2 — Menu bar icon visible (APP-02)**
Look at your macOS menu bar (top right area, near clock).
Expected: A waveform icon (SF Symbol "waveform") appears in the menu bar.

**Step 3 — Menu content**
Click the menu bar icon.
Expected: A dropdown appears with:
- Status line (Idle / waveform icon)
- "Never backed up" or a timestamp
- "Back Up Now" button
- "Launch at Login" toggle
- "Quit AbletonBackup"

**Step 4 — Manual backup trigger + icon state change (TRIG-03, APP-02)**
Click "Back Up Now" in the menu.
Expected:
- Menu bar icon changes from waveform to spinning arrows (arrow.triangle.2.circlepath) briefly
- Icon returns to waveform when done
- Menu now shows a "Last backup:" timestamp
- A macOS notification banner appears: "Backup Complete" with project name (NOTIF-01)

If a notification does NOT appear: check System Settings > Notifications > AbletonBackup and ensure alerts are allowed. The app requests permission at first launch.

**Step 5 — FSEvents save detection (TRIG-01, DISC-01)**
Open Finder and navigate to your Ableton Projects folder (typically ~/Documents/Ableton/Ableton Projects/).
Create or copy any .als file into that folder (or open Ableton and save a project there).
Wait up to 5 seconds.
Expected: Menu bar icon briefly shows spinning arrows, then returns to idle. A "Backup Complete" notification appears.

If no trigger fires: the watch folder discovery may have returned nil (Ableton Projects folder not at default location). Check if the folder exists at ~/Documents/Ableton/Ableton Projects/. If it doesn't exist, this is expected — Phase 3 settings UI adds manual folder configuration.

**Step 6 — Failure notification (NOTIF-02)**
This is tested indirectly: if you saw the icon go to error state (exclamationmark.triangle.fill) at any point, a failure notification should have appeared. If the app ran cleanly in Step 4, NOTIF-02 is covered by code review (the error path calls NotificationService.sendBackupFailure).

**Step 7 — Launch at Login toggle (APP-03)**
Click the "Launch at Login" toggle in the menu to enable it.
Expected:
- Toggle shows as enabled
- If "Approval required in System Settings" appears: click "Open System Settings…" and approve it there
- After approval (or if no approval needed): toggle stays enabled after you re-open the menu

Note: Rebooting to confirm actual login launch is optional for this checkpoint. The SMAppService API is well-tested; if register() succeeds without error and status == .enabled, it will work at login.
